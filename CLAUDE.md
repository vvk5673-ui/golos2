# Правила работы с проектом Golos 2.0

## Язык

- Всегда пиши на русском языке
- Комментарии в коде — на русском языке
- Названия переменных и функций — на английском (snake_case, Python-стиль)

## Что это за проект

- Голосовой ввод текста для Windows
- Горячая клавиша Alt+X, 40+ голосовых команд, трей-иконка, пользовательский словарь
- Два движка: Deepgram Nova-3 (облако, основной) и Vosk (оффлайн, запасной)
- Основной файл: voice_input.py (~700 строк)

## Проектная документация

- Перед началом работы прочитай project.md — описание проекта
- Перед началом задачи сверься с PLAN.md — актуальный план и статус
- Голосовые команды описаны в COMMANDS.md
- Ключевые слова для Deepgram — в keyterms.txt
- Пользовательский словарь — в dictionary.txt

## Стиль работы

- Двигаемся последовательно, шаг за шагом
- Каждый шаг согласовываем перед выполнением
- Не усложняй — решения должны быть простыми и понятными
- Объясняй свои решения простым языком

## Безопасность

- API-ключ Deepgram хранится в .env — НИКОГДА не хардкодить
- Файл .env всегда в .gitignore
- Не коммитить .env, ключи, токены

## Важные технические детали

- Deepgram подключается через прямой WebSocket (SDK v6 нестабилен)
- smart_format НЕ включать — мешает пунктуации на русском
- numerals у Deepgram НЕ поддерживает русский — используется своя конвертация чисел
- dictation НЕ использовать — не помогает
- utterance_end_ms минимум 1000 мс (меньше — HTTP 400)
- Нормализация аудио только для Vosk, НЕ для Deepgram

## Тестирование

- Запуск через ярлык "Golos 2.0" на рабочем столе, НЕ из терминала
- Перед тем как сказать что задача готова — проверь что всё работает
- После изменений — протестировать оба движка (Deepgram и Vosk)

## Git

- Коммить каждую завершённую задачу отдельно с описанием на русском
- Не коммить файлы с секретами (.env, ключи, токены)
- Перед коммитом проверь git status — не попало ли лишнее
- GitHub: vvk5673-ui/golos2

## Технические правила

- Код должен быть чистым и читаемым
- Не добавляй зависимости без согласования
- Не удаляй и не перемещай файлы без спроса
- Не используй платные API без согласования
- Разбивка на модули — пока отложена (700 строк не критично)
- Логирование в golos2.log с ротацией (макс 1 МБ, 2 бэкапа)
